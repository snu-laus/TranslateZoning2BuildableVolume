# TranslateZoning2BuildableVolume 이해 문서 (건축가 관점)

이 문서는 개발 지식이 없어도, 이 프로젝트가 **무엇을 넣어 무엇을 내는지**와 **중간에 어떤 판단을 하는지**를 이해할 수 있도록 정리한 설명서입니다.

---

## 1) 이 프로젝트가 하는 일 (한 줄 요약)

- 대지/도로 SHP 데이터를 넣으면, 정북사선 기준으로 **층별 건축 가능 면적**을 계산해 CSV로 내보냅니다.

---

## 2) 인풋(Input) 정의

### A. 공간 데이터 인풋

- `Parcels.shp` 계열 파일(필수)
  - 필지 경계(폐곡선)
  - PNU(필지 식별자), 지목, 용도지역 코드(A13), 공동주택 여부 등 속성
- (선택) 특정 필지만 계산하는 제한값
  - 예: 상위 N개만 테스트 계산

### B. 계산 조건 인풋

- 사선 계산 높이(배치 모드에서는 1층~7층, 층고 3m로 자동 반복)
- 사선 방식(`setback_type`)
  - `1`: 10m 이하 1.5m, 10m 초과는 높이 비례(기본 0.5배)
  - `2`: 10m 이하 1.5m, 10~17m 5m 고정, 17m 초과 높이 비례
- 기본 방향
  - 정북 방향 벡터(북쪽 기준)

### C. 내부 기본값(프로젝트 정책)

- 대상 필지: 일반주거지역 코드 `13/14/15`만 배치 계산
- 이웃 필지 사전 탐색거리: 300m
- 대상 필지 안쪽 오프셋: 1m
- 도로/비규제 용도 관련 20m 예외 규칙 포함

---

## 3) 아웃풋(Output) 정의

### A. 핵심 결과 CSV (`data/result/`)

각 행은 “한 필지의 한 층” 결과입니다.

주요 컬럼:
- `pnu`: 필지 번호
- `jimok`: 지목
- `landuse_code`, `landuse`: 용도지역 코드/명칭
- `lot_area_m2`: 원 필지 면적
- `lot_area_inward_1m_m2`: 1m 안쪽 보정 면적
- `floor`: 층(1~7)
- `height_m`: 층 높이(3,6,9,...,21m)
- `allowed_area_m2`: 해당 층 건축 가능 면적
- `base_segment_count`: 사선 기준선 개수

### B. QA(점검) CSV (`data/qa/`)

- 경고 대상 PNU 목록
- 20m 제외 규칙이 적용된 건
- 공동주택 중심선 보정이 적용된 건

즉, 계산 결과뿐 아니라 **“왜 제외/보정됐는지”**까지 추적 가능하게 기록합니다.

### C. Grasshopper 단일 실행 출력

- 사선 기준선 목록
- 건축 가능 경계선
- 시각화용 커터(Brep)

---

## 4) 중간 작동 로직(읽기 쉬운 중간언어 버전)

아래는 개발 코드가 아니라, **설계 업무 절차 문장**으로 바꾼 버전입니다.

1. SHP에서 필지/도로를 읽고, 필지 성격(용도지역, 공동주택 여부)을 정리한다.
2. 계산 대상은 일반주거지역(13/14/15) 필지로 한정한다.
3. 각 대상 필지마다 주변 이웃 필지를 모은다(먼 필지는 1차로 제외).
4. 대상 필지 경계 중에서 “북쪽 사선 영향이 시작되는 면(기준선 후보)”을 찾는다.
5. 각 기준선에 대해, 실제로 영향을 주는 맞은편/주변 경계를 찾아 사선 시작선을 만든다.
6. 다음 예외를 적용해 기준선을 정제한다.
   - 일조권 비규제 용도이고 거리 조건(20m 이상)이면 제외
   - 일반주거 공동주택의 중심선 케이스는 기준선 위치를 보정
7. 정제된 기준선을 기준으로 높이별 후퇴 거리(타입1/타입2 규칙)를 적용해 “잘라낼 띠(커터)”를 만든다.
8. 대상 필지에서 커터를 빼서, 남는 영역을 해당 높이의 건축 가능 경계로 본다.
9. 이 경계를 다시 “필지 안쪽 1m 보정 경계”와 겹쳐 최종 건축 가능 영역을 확정한다.
10. 최종 면적을 층별로 기록하고 CSV로 저장한다.

---

## 5) 중간 작동 로직(수도코드 버전)

```text
입력: SHP 경로, 사선방식(type), 층수 범위(1~7), 층고(3m)
출력: 층별 건축가능면적 CSV + QA CSV

SHP 로드
필지/도로 분류
대상필지 = 필지 중 landuse_code가 13/14/15 인 것

각 대상필지 target_lot 에 대해:
    주변필지 other_lots 수집 (bbox + 거리기반 사전 필터)

    기준선들 base_segments 계산:
        target_lot 경계에서 북향 영향 면 후보 추출
        후보별로 주변필지와의 노출 관계 분석
        예외규칙 적용:
            - 비규제용도 + 20m 조건 => 제외 표시(QA)
            - 공동주택 중심선 케이스 => 중심 보정(QA)
        너무 짧은 세그먼트 제거

    lot_region_inward = target_lot 경계를 안쪽 1m 오프셋
    만약 lot_region_inward 가 없으면:
        경고(QA) 기록
        허용면적 0으로 저장
        다음 필지로 이동

    floor = 1..7 반복:
        height = floor * 3m
        setback_distance = type 규칙으로 계산

        cutters 생성:
            각 base_segment를 setback_distance만큼 후퇴시켜 띠 생성

        buildable_raw = target_lot - cutters
        buildable_final = buildable_raw ∩ lot_region_inward

        allowed_area = 면적(buildable_final) (없으면 0)
        결과행 저장

결과 CSV 저장
QA CSV 저장
```

---

## 6) 건축 실무 해석 포인트

- 이 계산의 본질은 “법규 사선 때문에 실제로 못 쓰는 띠를 먼저 빼고, 남는 면적을 층별로 보는 방식”입니다.
- 그래서 같은 필지라도 높이가 올라가면 보통 `allowed_area_m2`가 줄어드는 경향을 보입니다.
- 타입1/타입2를 함께 돌리면, 정책안 차이에 따른 면적 변화를 같은 형식으로 비교할 수 있습니다.
- QA 파일은 결과 검증 시 매우 중요합니다. (특히 20m 제외/공동주택 중심선 보정 여부)
