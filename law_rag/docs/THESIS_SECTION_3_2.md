# 3.2 LLM + RAG 활용 법규 정보 추출 (개편본)

## 3.2.1 방법론
본 연구의 법규 정보 추출 모듈은 Retrieval-Augmented Generation(RAG) 구조를 채택하였다. 이 구조는 질의에 대해 법규 원문에서 관련 근거를 먼저 검색한 뒤, 검색 결과를 기반으로 답변을 생성함으로써 근거 추적성과 설명 가능성을 확보한다. 특히 본 연구의 목적은 단순 질의응답 정확도 향상이 아니라, **주어진 법규 검토 과제를 수행하기에 필요한 정보를 충분히 수집했는지 판단**하는 데 있다.

본 프로젝트의 생성 모델은 Ollama 환경에서 제공되는 후보 중 `llama3.1:8b`를 채택하였다. 해당 모델은 로컬 장비에서 안정적으로 구동 가능한 파라미터 규모를 가지면서도 한국어 질의응답 성능이 비교적 균형적이어서, 응답 품질과 추론 지연시간을 함께 만족시켰다. 또한 별도 API 의존 없이 반복 실험이 가능해 재현성과 운영 비용 측면에서 연구 목적에 적합하다고 판단하였다.

검색기는 TF-IDF, BM25, Dense Embedding, 그리고 페어 하이브리드 조합(TF-IDF+BM25, BM25+Dense, TF-IDF+Dense)을 지원한다. 하이브리드 점수는 두 검색기 점수를 정규화한 뒤 다음과 같이 결합하였다.

$$
S_{hybrid}(d,q)=\alpha\cdot \hat{S}_{A}(d,q)+(1-\alpha)\cdot \hat{S}_{B}(d,q),\quad \alpha\in[0,1]
$$

또한 질의 내 법규 신호(예: 제86조, 일조, 채광, 정북, 인접대지경계선, 높이제한)를 탐지해 검색 점수를 보정하였다. 본 구현의 청킹은 고정 길이 슬라이딩 윈도우(900자, overlap 120자) 방식으로 구성하였다.

## 3.2.2 준비도 중심 평가지표 설계
본 연구는 새 지표를 정의하기보다, 정보검색 및 분류에서 널리 사용되는 표준 지표를 법규 검토 목적에 맞게 적용하였다. 평가 목표는 다음 세 가지이다.

1. **완전성**: 필요한 조문·근거를 빠짐없이 찾았는가?
2. **부족 감지**: 자료가 부족하면 부족 항목을 식별했는가?
3. **근거 명시성**: 검색된 근거가 실제 판단에 연결 가능한가?

이에 따라 다음 지표를 사용하였다.

- **Recall@5**: 필수 근거 항목이 상위 5개 검색 결과에서 얼마나 회수되는지
- **Precision@5**: 상위 5개 검색 결과 중 실제로 유효 근거인 항목의 비율
- **MRR**: 첫 유효 근거가 얼마나 상위에 노출되는지
- **Readiness Accuracy**: “판단 가능/불가” 이진 분류 정확도

실험은 50개 법규 질의 평가셋으로 수행하였고, 이 중 일부 질의는 필수 첨부자료가 데이터셋에 없다고 가정한 부족 상황을 포함하여 준비도 판정의 정확성을 측정하였다.

모델 우위 판정은 임의 가중합 점수 대신 다음의 **게이트 + 순차 우선순위 규칙**으로 수행하였다. 먼저 Readiness Accuracy가 0.85 이상인 모델만 후보로 통과시키고(게이트), 통과 모델 간에는 Recall@5 > Precision@5 > MRR 순으로 우열을 판정하였다. 해당 기준은 본 연구의 목적(검토 가능 여부를 먼저 보장하고, 이후 근거 수집 완전성과 근거 품질을 비교)과 정합성이 높다.

## 3.2.3 실험 결과 및 해석
단일 검색기 결과는 다음과 같다.

- **TF-IDF**: Recall@5=0.872, Precision@5=0.636, Readiness Accuracy=0.880, MRR=0.743
- **BM25**: Recall@5=0.752, Precision@5=0.524, Readiness Accuracy=0.760, MRR=0.674
- **Dense Embedding**: Recall@5=0.702, Precision@5=0.444, Readiness Accuracy=0.700, MRR=0.617

페어 하이브리드의 최적 조합(\(\alpha=0.1\sim0.9\) 스윕 기준)은 다음과 같다.

- **TF-IDF+BM25**: \(\alpha=0.5\), Recall@5=0.788, Precision@5=0.528, Readiness Accuracy=0.780, MRR=0.693
- **BM25+Dense**: \(\alpha=0.4\), Recall@5=0.798, Precision@5=0.532, Readiness Accuracy=0.780, MRR=0.695
- **TF-IDF+Dense**: \(\alpha=0.8\), Recall@5=0.768, Precision@5=0.480, Readiness Accuracy=0.760, MRR=0.643

게이트 적용 결과 Readiness Accuracy 0.85를 충족한 모델은 TF-IDF 단일 방식(0.880)만이었으며, 따라서 최종 선택 모델은 **TF-IDF 단일 방식**으로 결정되었다. 즉, 현재 데이터와 질의 분포에서는 일부 하이브리드 조합이 특정 지표에서 개선을 보였으나, 연구의 1차 요구조건(판단 가능/불가 판정 신뢰성)을 만족하지 못해 최종 우위 모델로 채택되지 않았다.

## 3.2.4 선행연구 기반 설계 정합성
선행연구는 검색기 우위를 단일 모델 가정으로 판단하지 않고, lexical·dense·hybrid를 동일 조건에서 비교할 것을 권고한다(Thakur et al., 2021; Karpukhin et al., 2020). 본 연구는 이 원칙을 법규 검토 맥락에 맞게 적용하여, 표준 검색 지표(Recall@k, Precision@k, MRR)와 표준 분류 지표(Accuracy)를 함께 사용해 “찾기-판단 가능 여부 확인” 과정을 평가하였다.

요약하면, 본 모듈의 기여는 (1) 법규 근거 기반 RAG 구조 확립, (2) 다중 검색기 및 가중치 전수 탐색을 통한 재현 가능한 비교, (3) 법규검토 목적에 정합한 준비도 중심 평가 프레임 제시에 있다.
